# Smart Trike: Ultra Mega UI/UX Revamp Plan 🚀

This plan outlines the transformation of the Smart Trike application from a functional MVP to a premium, high-end mobile experience. We focus on "Not AI Slop" - meaning thoughtful, deliberate design choices that improve usability and aesthetic appeal.

## 1. Design Philosophy: "Modern Marinduque"
*   **Color Palette:** Shifting from basic blue to a sophisticated **Emerald & Slate** theme. Emerald represents the lushness of Marinduque, while Slate provides a clean, modern contrast.
*   **Surface Treatment:** Heavy use of **Glassmorphism** (translucent backgrounds with blur) for overlays and floating cards.
*   **Depth:** Moving beyond simple elevations to multi-layered, soft shadows.
*   **Typography:** Clean, geometric sans-serif (Inter/Roboto) with clear hierarchy and generous letter spacing.

## 2. Core Component Enhancements
### A. The "Mega" Button
*   Slightly larger hit areas.
*   Subtle gradient backgrounds.
*   Scale-on-press animations for tactile feedback.
*   Haptic feedback integration (where supported).

### B. Smart Inputs
*   Floating labels with smooth transitions.
*   Interactive validation states (glow effects for focus/error).
*   Clear-text icons and context-aware keyboards.

### C. Premium Cards
*   Soft rounded corners (24pt+).
*   Subtle borders instead of heavy shadows.
*   Integrated micro-animations.

## 3. Screen-Specific Revamps
### A. Authentication (Login/Register)
*   **Full-screen Background:** Use high-quality local scenery (Marinduque landmarks) with a dark overlay.
*   **One-tap Demo:** Keep the demo buttons but style them as premium "Quick Start" options.
*   **Smooth Entry:** Staggered fade-in animations for form elements.

### B. Passenger Dashboard
*   **Map Customization:** Implementation of a custom 'Silver' or 'Aubergine' map style.
*   **Floating Search:** A prominent, rounded "Where are you going?" bar that anchors to the top.
*   **Interactive Bottom Sheet:** A smooth, swipeable sheet for selecting tricycle types and viewing fare estimates.

### C. Driver Experience
*   **Duty Toggle:** A satisfying, oversized toggle for "Going Online".
*   **Actionable Notifications:** Large, easy-to-tap Accept/Reject cards with countdown timers.
*   **Route Visualization:** High-visibility paths for the current trip.

## 4. Functional "Super" Features
*   **Live Chat:** Real-time messaging with pre-defined "Quick Replies" (e.g., "I'm here", "Wait for me").
*   **Fare Transparency:** An interactive "Fare Explorer" that shows how the price is calculated based on TODA rules.
*   **Driver Profile Popovers:** View driver ratings and tricycle photos before the trip starts.

## 5. Implementation Roadmap
1.  **Phase 1 (The Foundation):** Update `theme.ts` and core components (`Button`, `Card`, `Input`).
2.  **Phase 2 (The Entry):** Complete overhaul of Login/Register screens.
3.  **Phase 3 (The Core):** Revamp Passenger and Driver dashboards.
4.  **Phase 4 (The Polish):** Add animations, micro-interactions, and the "Super" features.
